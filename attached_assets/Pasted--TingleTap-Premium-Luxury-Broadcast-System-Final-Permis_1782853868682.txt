# TingleTap – Premium Luxury Broadcast System (Final Permission Logic)

Implement a premium Broadcast System without breaking any existing functionality, Firebase architecture, moderation, chat, rooms, themes, or permissions.

## Permission Logic

RJ is **NOT** a role.

RJ is only a Badge/Permission assigned by Owner/Admin.

The user's original role (Registered User, VIP, Moderator, etc.) must remain unchanged.

The RJ Badge only unlocks RJ controls.

---

## Header Button

Add one premium animated SVG **Broadcast** button in the Header.

**Placement:**
Between the **Friend Request** button and the **Delete Chat** button.

This button is visible to **ALL users**, because everyone should be able to access broadcasts.

---

## Tab Visibility Rules

The Broadcast Panel contains **3 tabs**.

### Tab 1 — RJ Broadcast

For **RJ Badge Holders**
Show full controls:

- Go Live / End Broadcast
- Mic On/Off
- Invite Speaker
- Remove Speaker
- Accept/Reject Join Requests
- Listener Queue
- Broadcast Settings
- YouTube URL
- Play / Pause / Resume / Stop
- Song Controls
- Listener Count
- Broadcast Analytics

For **Normal Users**
Do NOT show RJ controls.

Instead show:

- Premium Broadcast Player
- RJ Avatar
- RJ Name
- RJ Badge
- LIVE Badge
- Broadcast Title
- Current Song
- Animated Premium Music SVG
- Live Listener Count
- Connected Speakers

Users can only listen.

---

### Tab 2 — Join Requests

Visible to everyone.

If a broadcast is live:

Users can:

- Request to Join
- Cancel Request

RJ sees:

- Pending Requests
- Accept
- Reject

Accepted users become temporary WebRTC voice co-hosts.

---

### Tab 3 — Public Broadcast

Visible to everyone.

Every registered user can:

- Start their own Audio Broadcast
- Public Broadcast
- Password Protected Broadcast

Each broadcast has its own premium player and listeners.

Keep architecture ready for future Video Broadcast support.

---

## Premium Broadcast Player

Display only ONE luxury player.

Never show YouTube thumbnails.

Instead use:

- Premium animated Music SVG
- RJ Avatar
- RJ Badge
- Broadcast Title
- Song Name
- Listener Count
- Live Timer
- Connected Speakers

Internally:

- WebRTC handles live voice.
- Embedded YouTube Player handles synchronized music.

Users should feel both are coming from one premium player.

---

## Music Synchronization

Only the RJ controls:

- YouTube URL
- Play
- Pause
- Resume
- Seek
- Song Change

Synchronize using Firebase:

- URL
- Playback Timestamp
- Play/Pause State

Automatically keep every listener synchronized.

---

## Voice System

Use WebRTC only for live voice.

Use Firebase Realtime Database for signaling.

Configure Google's free STUN server:

stun:stun.l.google.com:19302

Keep TURN support modular for future upgrades.
